Instructivo de instalacion en otra PC

Guia breve para instalar Tesoreria Santa Ana en una computadora nueva con Windows y conservar los datos actuales.

1. Que hay que llevar a la otra PC

  • La carpeta completa del proyecto AppJorge.
  • El archivo prisma/dev.db, porque ahi esta la base de datos actual.
  • El archivo .env.

No hace falta copiar node_modules ni .next, porque se regeneran.

2. Programas que hay que instalar

  1. Node.js
  2. Git opcional, solo si despues queres actualizar el proyecto desde repositorio

Se recomienda usar una version actual LTS de Node.js.

Verificacion

Abrir una terminal y ejecutar:

node -v
npm -v

3. Donde ubicar la carpeta

Copiar el proyecto, por ejemplo, en:

C:\Tesoreria\AppJorge

4. Ajuste obligatorio del archivo .env

Abrir .env y revisar DATABASE_URL. Si aparece una ruta absoluta de la PC original, cambiarla.

Dejarlo asi:

DATABASE_URL="file:./prisma/dev.db"
SESSION_COOKIE_NAME="tesoreria_session"
SESSION_TTL_HOURS="12"
APP_NAME="Tesoreria Santa Ana"
ADMIN_EMAIL="admin@tesoreria.local"
ADMIN_PASSWORD="Cambiar123!"
Importante: si no corregis DATABASE_URL, la app no va a encontrar la base en la nueva computadora.

5. Instalacion paso a paso

Abrir una terminal dentro de la carpeta del proyecto y ejecutar en este orden:

npm install
npm run prisma:generate
npm run prisma:push

6. Si la base ya viene copiada

Si copiaste prisma/dev.db, no hace falta importar datos. Con los comandos del punto anterior alcanza.

7. Si la base no viene copiada

La app va a arrancar vacia. En ese caso, despues de los comandos anteriores ejecutar:

npm run prisma:seed-admin

Eso crea el usuario administrador usando ADMIN_EMAIL y ADMIN_PASSWORD definidos en .env.

8. Como iniciar la aplicacion

npm run dev

La aplicacion queda disponible en:

http://localhost:3008

9. Como ingresar

  • Email: el valor de ADMIN_EMAIL
  • Clave: el valor de ADMIN_PASSWORD

10. Verificacion final

  1. Abrir http://localhost:3008
  2. Iniciar sesion
  3. Verificar que aparezcan empleados
  4. Verificar que aparezcan saldos y movimientos

11. Problemas comunes

Error por base de datos

Revisar que exista el archivo prisma/dev.db y que en .env figure:

DATABASE_URL="file:./prisma/dev.db"

Error porque no reconoce npm o node

Significa que Node.js no esta instalado o no quedo agregado al sistema. Cerrar y volver a abrir la terminal. Si sigue igual, reinstalar Node.js.

La app abre pero no hay datos

Probablemente no se copio prisma/dev.db. Hay que copiar ese archivo desde la PC original o volver a cargar los datos.

Error de dependencias

Ejecutar nuevamente:

npm install

12. Resumen corto

  1. Instalar Node.js
  2. Copiar la carpeta del proyecto
  3. Copiar prisma/dev.db
  4. Ajustar .env con DATABASE_URL="file:./prisma/dev.db"
  5. Ejecutar: 
    npm install
npm run prisma:generate
npm run prisma:push
npm run dev
  6. Abrir http://localhost:3008
